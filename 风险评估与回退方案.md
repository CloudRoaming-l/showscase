# 学生作品展示墙 - 风险评估与回退方案

## 一、风险管理体系

### 1.1 风险识别流程

```
识别 → 评估 → 规划 → 监控 → 响应
  │       │       │       │       │
  ▼       ▼       ▼       ▼       ▼
识别风险 评估影响 制定应对 持续监控 触发执行
```

### 1.2 风险评估矩阵

| 概率 \ 影响 | 低 (1-2) | 中 (3-4) | 高 (5) |
|------------|----------|----------|--------|
| **高 (80%+)** | 中风险 | 高风险 | **极高风险** |
| **中 (40-80%)** | 低风险 | 中风险 | **高风险** |
| **低 (<40%)** | 极低 | 低风险 | **中风险** |

### 1.3 风险等级定义

| 等级 | 说明 | 响应时间 | 处理策略 |
|------|------|----------|----------|
| 🔴 极高 | 可能导致项目失败 | 立即 | 必须消除 |
| 🔴 高 | 严重影响进度/质量 | 24h | 必须缓解 |
| 🟡 中 | 较大影响 | 72h | 需要缓解 |
| 🟢 低 | 轻微影响 | 1周 | 接受或监控 |
| ⚪ 极低 | 可忽略影响 | - | 接受 |

| 项目 | 内容 |
|------|------|

| **触发条件** | 使用 React 18 的 Suspense、useTransition 等特性时 |

**预防措施**：
```json
// package.json 中锁定版本
"dependencies": {
  "react": "^18.2.0",
  "react-dom": "^18.2.0",
  "vite": "^5.0.0"
}
```

**应对措施**：
1. 如果遇到兼容性问题，降级到 React 17.2
2. 避免使用不稳定的实验性特性
3. 等待官方修复后再升级

**回退方案**：
- 降级 React 到 17.x 版本
- 保持现有功能正常

```javascript
// 1. 图片压缩配置 (Sharp)
const compressionOptions = {
```

### 2.3 安全风险

#### 风险 5：文件上传漏洞

| 项目 | 内容 |
|------|------|
| **风险描述** | 用户上传恶意文件（Webshell、木马等） |
| **发生概率** | 🟢 低 (10%) |
| **影响程度** | 🔴 极高（服务器被完全控制） |
| **风险等级** | 🔴 极高风险 |
| **触发条件** | 上传功能上线且未做严格校验 |

**预防措施**：
```javascript
// Multer 配置
const storage = multer.diskStorage({
  destination: '/tmp/uploads',
  filename: (req, file, cb) => {
    const uniqueSuffix = Date.now() + '-' + Math.round(Math.random() * 1E9);
    cb(null, uniqueSuffix + path.extname(file.originalname));
  }
});

const upload = multer({
  storage,
  limits: {
    fileSize: 5 * 1024 * 1024, // 5MB
    files: 1
  },
  fileFilter: (req, file, cb) => {
    // 严格的白名单
    const allowedTypes = ['image/jpeg', 'image/png', 'image/webp'];
    if (!allowedTypes.includes(file.mimetype)) {
      return cb(new Error('Invalid file type'), false);
    }
    // 检查文件扩展名
    const ext = path.extname(file.originalname).toLowerCase();
    const allowedExts = ['.jpg', '.jpeg', '.png', '.webp'];
    if (!allowedExts.includes(ext)) {
      return cb(new Error('Invalid extension'), false);
    }
    cb(null, true);
  }
});
```

**应对措施**：
1. 文件类型 MIME 检查
2. 文件扩展名校验
3. 文件大小限制
4. 文件内容扫描（ClamAV）
5. 上传后移动到 OSS（不存储在本地）

**回退方案**：
- 立即关闭上传功能
- 删除已上传的可疑文件
- 审查上传日志

#### 风险 6：JWT Token 安全

| 项目 | 内容 |
|------|------|
| **风险描述** | Token 被窃取、伪造、重放攻击 |
| **发生概率** | 🟢 低 (15%) |
| **影响程度** | 🔴 高（账户被盗用） |
| **风险等级** | 🟡 中风险 |

**预防措施**：
```javascript
// JWT 配置
const jwtConfig = {
  secret: process.env.JWT_SECRET, // 至少 256 位
  expiresIn: '1h',
  refreshExpiresIn: '7d',
  algorithm: 'HS256'
};

// 使用 Refresh Token
const accessToken = jwt.sign(
  { userId: user._id, role: user.role },
  jwtConfig.secret,
  { expiresIn: jwtConfig.expiresIn }
);

const refreshToken = jwt.sign(
  { userId: user._id, type: 'refresh' },
  jwtConfig.secret,
  { expiresIn: jwtConfig.refreshExpiresIn }
);

// Token 黑名单（退出登录时）
const blacklist = new Set();
const isBlacklisted = blacklist.has(token);

// HttpOnly Cookie 存储
res.cookie('token', token, {
  httpOnly: true,  // 防止 XSS
  secure: true,    // 仅 HTTPS
  sameSite: 'strict' // CSRF 防护
});
```

**应对措施**：
1. Token 短期有效（1小时）
2. Refresh Token 长期有效但可撤销
3. 登录异常检测
4. IP 地址绑定（可选）

**回退方案**：
- 强制所有用户重新登录
- 撤销所有 Refresh Token
- 调查安全日志

|------|----------|----------|------|
| 进度风险 | 任务完成率 | < 80%/周 | 增加资源 |
| 性能风险 | Lighthouse 分数 | < 80 | 优化性能 |
| 安全风险 | 漏洞数量 | > 0 | 立即修复 |
| 用户风险 | 使用率 | < 50% | 改进培训 |

### 7.2 预警机制

```javascript
// 风险监控系统（示例）
class RiskMonitor {
  constructor() {
    this.thresholds = {
      progress: 0.8,
      performance: 80,
      errors: 0,
      downtime: 0
    };
  }

  checkProgress(completed, total) {
    const rate = completed / total;
    if (rate < this.thresholds.progress) {
      this.alert('进度落后', `完成率 ${rate * 100}% < 80%`);
    }
  }

  checkPerformance(lighthouseScore) {
    if (lighthouseScore < this.thresholds.performance) {
      this.alert('性能不达标', `Lighthouse ${lighthouseScore} < 80`);
    }
  }

  alert(type, message) {
    // 发送预警通知
    console.error(`[${type}] ${message}`);
    // 可以集成钉钉/微信通知
  }
}
```
