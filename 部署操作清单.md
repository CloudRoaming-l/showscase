# kidsshowcode.cn Docker 部署操作清单（超详细版）

> 每一步都标明了：在哪个目录、执行什么命令、执行完应该看到什么结果。
> 照着从上往下做就行。整个过程预计 20-30 分钟。

---

## 第 0 步：在本地电脑把代码推到 GitHub

**在哪个目录**：`D:\MyApp_3`（你的本地项目根目录）

```bash
git add .
git commit -m "修复 Docker 部署全部问题 + 迁移脚本"
git push github main
```

> 推完之后，服务器才能拉到最新代码。

## 第 2 步：安装 Docker（如果已经装过就跳过）

**在哪个目录**：`/root`（任何目录都行，这是系统级操作）

> ```bash

## 第 8 步：迁移 MongoDB 数据到 Docker 容器

**在哪个目录**：`/www/wwwroot/showscase`（项目根目录）

```bash
# 给迁移脚本执行权限
chmod +x scripts/migrate-mongodb.sh

# 执行迁移
./scripts/migrate-mongodb.sh
```

脚本会自动完成 4 件事：
1. 从裸机 MongoDB 导出数据（mongodump）
2. 导入到 Docker MongoDB 容器（mongorestore）
3. 复制上传的图片/文件（uploads/）到 Docker 卷
4. 验证数据完整性

执行完后应该看到：
```
✅ 迁移完成！
```

手动验证数据数量：
```bash
docker exec showcase-mongodb mongosh \
    -u admin -p 'Kidsshow2026!Secure' \
    --authenticationDatabase admin \
    --eval "use student-showcase; print('photos:', db.photos.countDocuments()); print('students:', db.students.countDocuments()); print('scratch:', db.scratchprojects.countDocuments()); print('users:', db.users.countDocuments())"
```

> 数字应该和你原来数据库一致。比如 photos: 40+, students: 8, users: 1。

> 为什么先不启动 Nginx？因为 Nginx 容器要占用 80 和 443 端口，但现在宝塔 Nginx 还在用着这两个端口。等确认后端前端都正常了，再切换。

**在哪个目录**：`/www/wwwroot/showscase`（项目根目录）

```bash
# 只构建并启动后端和前端（首次构建约需 3-5 分钟）
```

```bash
#   showcase-backend    running
#   showcase-frontend   running
```

检查后端日志（确认 MongoDB 连接成功）：
```bash
docker compose logs backend --tail 20
```

> 日志里应该看到类似：
> ```
> ✅ MongoDB 已连接: mongodb://admin:...@mongodb:27017/student-showcase?authSource=admin
> 🚀 服务启动于 http://localhost:5001
> ```

测试后端 API 是否正常响应：
```bash
# 在服务器内部测试后端

# 应该返回 {"status":"ok",...} 之类的 JSON
```

## 第 10 步：停止旧服务，切换到 Docker Nginx

> 这一步会有几秒钟的短暂中断。建议在用户访问量少的时候做。

**在哪个目录**：`/www/wwwroot/showscase`（项目根目录）

### 10.1 停止 PM2 后端（Docker 后端已经接替它了）


### 10.3 确认端口已释放

```bash
netstat -tlnp | grep -E ':80|:443'
# 应该没有任何输出，或者只显示 docker-proxy
```

### 10.4 启动 Docker Nginx 容器

```bash
docker compose up -d nginx
```

### 10.5 确认所有 4 个容器都在运行

```bash
docker compose ps
# 应该看到 4 个容器全部 running：
#   showcase-mongodb    running (healthy)
#   showcase-backend    running
#   showcase-frontend   running
#   showcase-nginx      running
```

## 第 12 步：收尾工作

**在哪个目录**：`/www/wwwroot/showscase`（项目根目录）

### 12.1 禁用宝塔 Nginx 开机自启（防止重启后两个 Nginx 抢端口）

```bash
systemctl disable nginx
```

### 12.2 设置 Docker 开机自启

```bash
systemctl enable docker
```

### 12.3 设置 SSL 证书自动续期（宝塔续期后自动同步到 Docker）

```bash
# 创建自动续期脚本
cat > /www/wwwroot/showscase/scripts/renew-ssl.sh << 'EOF'
#!/bin/bash
# 宝塔续期 Let's Encrypt 证书后，自动复制到 Docker 目录并重启 nginx 容器
cp /www/server/panel/vhost/cert/kidsshowcode.cn/fullchain.pem /www/wwwroot/showscase/nginx/ssl/cert.pem
cp /www/server/panel/vhost/cert/kidsshowcode.cn/privkey.pem /www/wwwroot/showscase/nginx/ssl/key.pem
docker restart showcase-nginx
echo "[$(date)] SSL 证书已更新并重启 nginx 容器"
EOF

chmod +x /www/wwwroot/showscase/scripts/renew-ssl.sh
```

然后去宝塔面板添加定时任务：
- 宝塔面板 → 计划任务 → 添加任务
- 任务类型：Shell 脚本
- 任务名称：SSL 证书同步到 Docker
- 执行周期：每月 1 号
- 脚本内容：`/www/wwwroot/showscase/scripts/renew-ssl.sh`

### 12.4 清理临时文件

```bash
rm -rf /tmp/mongo-dump
```

## 第 13 步：排查问题（如果验证不通过）

**在哪个目录**：`/www/wwwroot/showscase`（项目根目录）

### 查看所有容器日志

```bash
docker compose logs --tail 30
```

### 端口被占用

```bash
# 查看谁占了 80/443
netstat -tlnp | grep -E ':80|:443'

# 如果是宝塔 Nginx 没停干净
systemctl stop nginx

# 然后重启 Docker nginx
docker compose up -d nginx
```

### 后端连不上 MongoDB

```bash
# 检查 .env 里的密码是否正确
cat .env | grep MONGO_PASSWORD

# 确认 MongoDB 容器健康
docker compose ps mongodb

# 看后端报什么错
docker compose logs backend --tail 30
```

### 前端页面空白或报错

```bash
# 检查前端容器日志
docker compose logs frontend --tail 20

# 检查前端容器内文件是否存在
docker exec showcase-frontend ls /usr/share/nginx/html/
# 应该看到 index.html 和 assets/ 目录
```

### 数据没迁移成功

```bash
# 重新运行迁移脚本
./scripts/migrate-mongodb.sh
```

## 第 14 步：回滚方案（万一搞不定，回到原来的部署）

**在哪个目录**：`/www/wwwroot/showscase`（项目根目录）

```bash
# 1. 停止所有 Docker 容器
docker compose down

# 2. 启动宝塔 Nginx
systemctl start nginx
systemctl enable nginx

# 3. 启动 PM2 后端
cd /www/wwwroot/showscase
pm2 start backend/src/app.js --name showcase-backend
pm2 save

# 4. 验证
curl -I https://kidsshowcode.cn
```

> 原来的数据没有受影响——迁移时是从裸机导出复制到 Docker，
> 裸机 MongoDB 的原始数据还在。回滚后直接用原来的就行。

## 后续更新流程（以后改了代码重新部署）

**在哪个目录**：`/www/wwwroot/showscase`（项目根目录）

```bash
# 拉取最新代码
git pull github main

# 重新构建并启动（只重建有变更的容器）
docker compose up -d --build

# 查看状态
docker compose ps
```

> ⚠️ **如果更新后前端出现 404、JS/CSS 加载失败、或页面空白**，很可能是 Docker 构建缓存没刷新 Vite 生成的 `index.html`。
> 这时需要清理缓存后重新构建：
> ```bash
> docker compose down
> docker compose build --no-cache
> docker compose up -d
> ```
>
> 也可以使用项目里的一键脚本：
> ```bash
> ./deploy-docker.sh update
> ```
