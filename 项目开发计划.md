## 五、质量保障

### 代码质量

- ESLint 代码规范检查
- Prettier 代码格式化
- Git Commit 规范（Conventional Commits）
- Code Review（代码审查）
- 单元测试覆盖率 > 80%

### 测试策略

| 测试类型 | 覆盖内容 | 工具 |
|----------|----------|------|
| 单元测试 | 组件、函数、逻辑 | Jest / React Testing Library |
| 集成测试 | API 接口、模块交互 | Supertest / Postman |
| E2E 测试 | 关键用户流程 | Playwright / Cypress |
| 性能测试 | 页面加载、响应时间 | Lighthouse / JMeter |
| 安全测试 | 注入、XSS、CSRF | OWASP ZAP |

### 代码审查清单

- [ ] 代码符合 ESLint 规范
- [ ] 有适当的单元测试
- [ ] 没有明显的性能问题
- [ ] API 响应格式正确
- [ ] 错误处理完善
- [ ] 注释清晰明了
- [ ] 没有敏感信息泄露
