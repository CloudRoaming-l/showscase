# 学生作品展示墙 - 技术架构方案

## 一、项目定位

**项目名称**：少儿编程学生作品展示系统  
**项目类型**：全栈 Web 应用  
**核心目标**：为 500 人规模少儿编程校区提供数字化作品展示与管理平台  
**目标用户**：校区教师、学生家长、潜在客户、校区管理者

### 2.1 前端技术栈

| 技术 | 选择 | 版本 | 说明 |

## 四、数据库设计

### 4.1 数据模型

#### Photo（作品）
```javascript
{
  _id: ObjectId,
  title: String,              // 作品名称
  description: String,        // 作品描述
  category: ObjectId,        // 关联分类
  student: ObjectId,           // 关联学生
  teacher: ObjectId,           // 上传教师
  images: [{                  // 图片数组（支持多图）
    url: String,              // OSS 地址
    width: Number,
    height: Number,
    thumbnail: String         // 缩略图
  }],
  tags: [String],             // 自定义标签
  isFeatured: Boolean,        // 是否精选
  isAward: Boolean,           // 是否获奖
  awardInfo: {               // 获奖信息
    competition: String,
    level: String,           // 市级/省级/国家级
    rank: String,            // 一等奖/二等奖
    year: Number
  },
  viewCount: Number,          // 浏览次数
  likeCount: Number,          // 点赞次数
  shareCount: Number,         // 分享次数
  status: String,             // draft/published/archived
  publishedAt: Date,          // 发布时间
  createdAt: Date,
  updatedAt: Date
}
```

#### Student（学生）
```javascript
{
  _id: ObjectId,
  name: String,               // 学生姓名
  avatar: String,              // 头像
  grade: String,              // 年级
  className: String,           // 班级
  phone: String,               // 家长电话（隐私）
  courses: [ObjectId],        // 学习的课程
  parentName: String,          // 家长姓名
  joinDate: Date,              // 入学日期
  photos: [ObjectId],         // 作品列表
  stats: {
    totalPhotos: Number,
    awards: Number,
    projects: Number
  },
  status: String,             // active/inactive
  createdAt: Date,
  updatedAt: Date
}
```

#### Category（分类）
```javascript
{
  _id: ObjectId,
  name: String,               // 分类名称
  slug: String,               // URL 友好名称
  description: String,
  icon: String,               // 图标
  color: String,              // 主题色
  order: Number,              // 排序
  count: Number,              // 该分类作品数量
  isActive: Boolean,
  createdAt: Date,
  updatedAt: Date
}
```

#### User（用户/教师）
```javascript
{
  _id: ObjectId,
  username: String,
  password: String,            // 加密存储
  name: String,                // 真实姓名
  role: String,                // admin/teacher
  phone: String,
  email: String,
  avatar: String,
  permissions: [String],       // 权限列表
  status: String,              // active/inactive
  lastLogin: Date,
  createdAt: Date,
  updatedAt: Date
}
```

#### Share（分享记录）
```javascript
{
  _id: ObjectId,
  photo: ObjectId,             // 被分享的作品
  student: ObjectId,           // 作品作者
  shareCode: String,           // 分享码
  platform: String,           // wechat/moments/qrcode
  viewerIP: String,
  createdAt: Date
}
```

## 五、API 接口设计

### 5.1 公开接口

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| GET | /api/photos | 获取作品列表（分页、筛选） | 否 |
| GET | /api/photos/:id | 获取作品详情 | 否 |
| GET | /api/photos/featured | 获取精选作品 | 否 |
| GET | /api/categories | 获取分类列表 | 否 |
| GET | /api/students/:id | 获取学生信息 | 否 |
| GET | /api/students/:id/photos | 获取学生作品 | 否 |
| GET | /api/stats/public | 公开统计数据 | 否 |

### 5.2 用户接口（需登录）

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | /api/auth/login | 用户登录 | 否 |
| POST | /api/auth/logout | 用户登出 | 是 |
| GET | /api/auth/me | 获取当前用户 | 是 |
| POST | /api/photos/:id/like | 点赞作品 | 是 |
| DELETE | /api/photos/:id/like | 取消点赞 | 是 |
| POST | /api/photos/:id/share | 分享作品 | 是 |
| GET | /api/users/favorites | 获取收藏列表 | 是 |
| POST | /api/users/favorites | 添加收藏 | 是 |

### 5.3 管理接口（需管理员权限）

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | /api/admin/photos | 创建作品 | 管理员 |
| PUT | /api/admin/photos/:id | 更新作品 | 管理员 |
| DELETE | /api/admin/photos/:id | 删除作品 | 管理员 |
| POST | /api/admin/photos/batch | 批量创建 | 管理员 |
| POST | /api/admin/upload | 文件上传 | 管理员 |
| GET | /api/admin/students | 学生列表 | 管理员 |
| POST | /api/admin/students | 创建学生 | 管理员 |
| GET | /api/admin/stats | 管理统计数据 | 管理员 |
| GET | /api/admin/logs | 操作日志 | 管理员 |

## 八、性能优化

### 8.1 前端优化

- React.lazy + Suspense 代码分割
- 图片懒加载
- 虚拟列表（大量数据）
- Service Worker 缓存
- 骨架屏加载
- Gzip 压缩

### 8.2 后端优化

- 数据库索引优化
- Redis 缓存热点数据
- 图片压缩和 CDN
- API 响应缓存
- 数据库连接池

### 8.3 缓存策略

| 数据类型 | 缓存时间 | 缓存位置 |
|----------|----------|----------|
| 分类列表 | 1小时 | Redis |
| 作品列表 | 5分钟 | Redis |
| 统计数据 | 10分钟 | Redis |
| 用户会话 | 30分钟 | Redis |
| 静态资源 | 30天 | CDN |
