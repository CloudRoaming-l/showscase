# 学生作品展示墙 - 全栈开发环境配置指南

## 一、必须安装的软件

### 1. Git（版本控制）
- **下载地址**：https://git-scm.com/download/win
- **版本**：选择 64-bit Git for Windows Setup
- **安装后验证**：打开 PowerShell，输入 `git --version`

### 2. VS Code（代码编辑器）
- **下载地址**：https://code.visualstudio.com/
- **推荐插件**：
  - Live Server（本地开发服务器）
  - Prettier（代码格式化）
  - ESLint（代码检查）
  - Auto Rename Tag（HTML标签自动补全）
  - Path Intellisense（路径补全）

## 二、推荐安装的工具

### 1. Node.js（已安装，可跳过）
- **当前版本**：v24.11.1
- **建议版本**：v20 LTS（更稳定）
- **下载地址**：https://nodejs.org/

### 2. MongoDB（数据库）
- **社区版**：https://www.mongodb.com/try/download/community
- **版本**：6.0+
- **可选**：使用 MongoDB Atlas（云数据库，免费额度）

### 3. Docker（可选，用于容器化部署）
- **下载地址**：https://www.docker.com/products/docker-desktop/
- **用途**：简化部署和环境配置

## 三、账号准备

### 1. GitHub/Gitee
- 注册 GitHub 账号：https://github.com/
- 用于代码托管和版本控制

### 2. 云服务（部署用）
- **阿里云 OSS**：图片存储
- **腾讯云 COS**：图片存储（备选）
- **Vercel/Netlify**：静态网站部署（免费）
- **Railway/Render**：后端部署（免费额度）

## 四、项目初始化命令

```bash
# 1. 克隆项目（如果已有仓库）
git clone <仓库地址>
cd student-showcase

# 2. 安装前端依赖
npm install

# 3. 安装后端依赖
cd server
npm install

# 4. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，填入数据库和API密钥

# 5. 启动开发服务器
npm run dev
```

## 五、环境检查清单

- [ ] Git 已安装并配置（git --version）
- [ ] VS Code 已安装并配置好插件
- [ ] Node.js 版本正确（node -v）
- [ ] MongoDB 已安装或拥有 Atlas 账号
- [ ] 已注册 GitHub/Gitee 账号
- [ ] 具备云存储服务账号（OSS/COS）

## 六、常见问题

### Q1: npm install 失败
```bash
# 清除缓存后重试
npm cache clean --force
rm -rf node_modules
npm install
```

### Q2: 端口被占用
```bash
# Windows 查看端口占用
netstat -ano | findstr :3000
taskkill /PID <进程ID> /F
```

### Q3: MongoDB 连接失败
- 检查 MongoDB 服务是否启动
- 检查连接字符串是否正确
- 确认网络连接和防火墙设置
